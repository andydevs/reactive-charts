## Project Overview

A React-based SVG chart component library. Currently implements a pie chart with a matching loading skeleton. No external charting libraries — all geometry and SVG path generation is done in TypeScript.

## Monorepo Structure

Yarn workspaces with two areas:

- `packages/components/` — the publishable library (TypeScript + React)
- `examples/webpack-site/` — example app demonstrating the library (React, Webpack)

## Commands

Run from the repo root:

```bash
yarn start                   # watch-build the library + serve the example site (development)
yarn build                   # build both library and example site for production

yarn build:lib               # build library only (tsc)
yarn build:lib:watch         # build library in watch mode
yarn serve:site:webpack      # serve example site only
```

No test or lint commands are configured yet.

## Architecture

### Library (`packages/components/src/`)

**Public API** — exported from `index.ts`:
- `PieChart` — interactive SVG pie chart
- `PieStencil` — loading skeleton (rotating gradient circle, uses SVG `<animateTransform>`)
- `PieCategory`, `PieStyle` — TypeScript interfaces for data and configuration

**Internal modules:**
- `PieChart/slice.ts` — data transformation pipeline (`toSlices`): sorts, normalizes values to angles, optionally collapses small slices into an "other" bucket
- `PieChart/label.ts` — label position calculation from slice midpoint angles
- `PieChart/types.ts` — `PieCategory { label, color, value }` and `PieStyle` (SVG dimensions, radii, gaps, corner radius, etc.)
- `support/point.ts` — polar coordinate math helpers (angle/radius → x/y)

### Component patterns

- **Styling via `PieStyle` object** — all visual properties (dimensions, radii, gaps, padding, animation) are controlled through a single config object, not CSS or individual props
- **`useMemo` for geometry** — slice and label calculations are memoized to avoid recomputation on unrelated renders
- **Loading pattern** — caller holds a loading boolean and renders `PieStencil` or `PieChart` conditionally (see `examples/webpack-site/src/app/index.jsx`)

### Data flow

```
PieCategory[] + PieStyle  →  toSlices()  →  SVG path strings  →  <PieChart />
```

`toSlices()` chain: sort by value → take top N → normalize to 0–2π angles → collapse small angles → return `Slice[]` with start/end angles.

### SVG generation

All paths are generated by hand (no D3 or similar). Corner radius on slice edges is approximated using arc commands. Labels are rendered as separate `<text>` elements positioned independently of the slice paths.
